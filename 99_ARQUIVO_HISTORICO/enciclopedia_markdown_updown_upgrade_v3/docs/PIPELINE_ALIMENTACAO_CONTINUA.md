# 🔄 Pipeline de alimentação contínua

## Fluxo rápido

1. Estudar material.
2. Fazer notas próprias.
3. Rodar prompt de transformação em Markdown.
4. Salvar arquivo em `07_Estudos_Markdown/content/<tema>/nome.md`.
5. Registrar no `07_Estudos_Markdown/registry.json`.
6. Atualizar `data/catalogo.json` somente se o estudo virar módulo/hub relevante.
7. Criar produtos derivados: card, flashcard, questões TEMI, checklist ou calculadora.
8. Conectar no mapa vivo quando o tema for importante.

## Status sugeridos

- `inbox`: recebido, ainda não revisado.
- `rascunho`: já estruturado, mas precisa revisar.
- `revisado`: conferido clinicamente.
- `pronto`: pronto para uso público.
- `atualizar`: precisa revisão por nova evidência.
- `template`: modelo operacional.

## Versionamento

- `v0.1`: primeira síntese.
- `v0.2`: acrescentadas condutas/algoritmos.
- `v0.5`: revisão clínica parcial.
- `v1.0`: versão pronta.
- `v1.1+`: atualizações menores.
- `v2.0`: reestruturação importante.
